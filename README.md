### ОПИС
UpdAter - програма для зручного, швидкого та автоматичного оновлення великої кількості перекладів на різні ігри. Для авторів перекладів це зменшення необхідного часу на встановлення оновлення і деяка автоматизація. Для звичайних гравців це змога завжди мати останні версії перекладів для улюблених ігор.

### ВСТАНОВЛЕННЯ Й ВИДАЛЕННЯ
Завантажте останній реліз [звідси](https://github.com/MaksymKurny/UpdAter/releases). Розпакуйте архів в зручне місце, запустіть файл "UpdAter.exe". Для видалення достатньо видалити всі файли.
Якщо ви хочете перенести / зберегти список доданих перекладів - скопіюйте старий файл "settings.json"(створиться після додавання першого перекладу) до теки щойно видобутої програми.

### ВИКОРИСТАННЯ ПРОГРАМИ
Для додавання нового перекладу натисніть кнопку "Додати ще" й відповідні рядки в формі.

1. Як отримати посилання на файл
* Для гугл-посилань: потрібне ПРЯМЕ посилання на файл. Якщо посилання на переклад веде в теку на гугл-диску то його можна отримати, клацнувши СКМ (середня кнопка миші / коліщатко) по потрібному файлу. Це відкриє нову вкладку, в адресі якої буде пряме посилання (воно починатиметься на httрs://drive.google.com/file/d/).
* Для GitHub: клацнути зелену кнопку "Code", у спадному меню клацнути ПКМ по рядку "Download ZIP", вибрати пункт "Копіювати посилання"  
![image](https://github.com/user-attachments/assets/621da7be-aa13-4861-a407-8ff23992a32e)

* Посилання з nexus.mods чи інших джерел - не підтримуються / можуть не працювати належним чином.

3. Далі вкажіть шлях до теки в яку має завантажуватись файл.

4. Для ігор Steam працює авто-підхоплення назви, іконки й банера (через додаткову кнопку "Заповнити", вона стане активною якщо шлях містить папки стіма). Ігри з інших лаунчерів які не мають підтримки автозаповнення потрібно заповнити власноруч, іконка і банер необов'язкові. 

5. Якщо всі обов'язкові поля будуть заповнені коректними даними(посилання починається з http, існуючий шлях до теки) то кнопка зберегти стане активною і після натискання переклад буде доданий то списку всіх перекладів.


З додаткових можливостей в меню перекладу є:
- Змінювати внесені раніше дані;
- Додавати в список для вибіркового оновлення кількох ігор одночасно кнопкою оновити список;
- Закріпити переклад в горі списку;
- Відкрити теку з файлами перекладу;
- Видалити переклад з списку (не видаляє завантажені файли і сам переклад).


### ІНША ІНФОРМАЦІЯ
> [!NOTE]
> Zip архіви будуть автоматично розпаковані в вибрану теку, майте це на увазі. Якщо потрібно зробити додаткові дії для встановлення файлів з архіву можете скористатись опцією в меню для швидкого відкривання теки.

> [!TIP]
> Програма також підтримує приховане автоматичне оновлення. для цього достатньо додати в параметри запуску ярлика наступний текст:
> ...\UpdAter.exe --hidden --game="назва гри 1;назва гри 2;назва гри ..."
> ![image](https://github.com/user-attachments/assets/7b2252c5-f093-4956-87f9-634d7e00638d)  
> Назви мають відповідати назвам які ви поставили в програмі. Якщо все виконано вірно при спробі запустити ярлик він запустить програму у фоновому режимі а після закінчення оновлення закриє її.

До уваги волонтерів-українізаторів: щоб програма працювала з вашими перекладами - залийте їх на Google-drive чи GitHub. Якщо потрібно оновлювати декілька файлів краще завантажуйте zip архіви. Лише в такому випадку програма зможе нормально їх опрацьовувати. Програма працює по принципу завантажив в 1 папку всі потрібні файли з заміною, тому якщо потрібна якась складна система з додаванням файлів в різні папки то краще за все продумати це наперед. 

Обов'язково переконайтесь, що будова вашого архіву ПРАВИЛЬНА й переклад працюватиме після видобутку в потрібну теку гри, створіть README з вказівками про додаткові дії з використанням UpdAter (якщо в них є потреба для вашого перекладу).

Ми не можемо врахувати всі нюанси та передбачити всі ситуації для різних проєктів - можуть знайтись ігри, котрі не вдасться оновлювати через UpdAter. Але ми постарались створити максимально просту платформу для поширення й зручного встановлення українізаторів. У разі виникнення багів чи проблем: постарайтесь точно відтворити послідовність дій, які їх викликають та повідомте нас в коментарях посібника Steam (посилання в кінці) - чи у відповідному розділі програми на GitHub.
